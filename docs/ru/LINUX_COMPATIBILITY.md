# Отчет о совместимости с Linux для NeoZorK3 Arbitrage Bot

## Обзор

Этот документ предоставляет комплексный анализ совместимости NeoZorK3 Solana Arbitrage Bot с системами Linux, протестированной с использованием нативной системы контейнеров Apple.

## Тестовая среда

- **Система контейнеров**: Apple native container (не Docker)
- **Дистрибутив Linux**: Ubuntu 22.04 LTS
- **Архитектура**: ARM64 (aarch64)
- **Компилятор**: GCC 11.4.0
- **Дата тестирования**: 9 августа 2024

## Протестированные зависимости

### ✅ Успешно проверенные зависимости

1. **Boost Libraries**
   - Boost.ASIO (сетевой I/O)
   - Boost.Thread (многопоточность)
   - Boost.System (системные утилиты)
   - Версия: 1.74.0

2. **OpenSSL**
   - Поддержка SSL/TLS
   - Криптографические функции
   - Версия: 3.0.2

3. **CMake**
   - Система сборки
   - Версия: 3.22.1

4. **nlohmann/json**
   - Парсинг и сериализация JSON
   - Версия: 3.10.5

5. **Стандартные C++ библиотеки**
   - Поддержка стандарта C++20
   - STL контейнеры и алгоритмы

## Процесс сборки

Бот был успешно собран с использованием следующего процесса:

```bash
# Установка зависимостей
apt-get update
apt-get install -y build-essential cmake libboost-all-dev libssl-dev libgtest-dev git nlohmann-json3-dev

# Конфигурация с CMake
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release

# Сборка проекта
make -j4
```

## Инструкции по развертыванию

### Для систем Ubuntu/Debian

```bash
# Обновление системы
sudo apt-get update

# Установка зависимостей
sudo apt-get install -y build-essential cmake libboost-all-dev libssl-dev libgtest-dev git nlohmann-json3-dev

# Клонирование и сборка
git clone <repository-url>
cd NeoZorK3
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)

# Запуск бота
./solana_arbitrage_bot --help
```

### Для систем CentOS/RHEL

```bash
# Установка зависимостей
sudo yum groupinstall -y "Development Tools"
sudo yum install -y cmake boost-devel openssl-devel gtest-devel git

# Процесс сборки такой же, как выше
```
