# Отчет о совместимости с Alpine Linux для NeoZorK3 Arbitrage Bot

## Обзор

Этот документ предоставляет комплексный анализ совместимости NeoZorK3 Solana Arbitrage Bot с системами Alpine Linux, которые используют musl libc вместо glibc.

## Тестовая среда

- **Система контейнеров**: Apple native container (не Docker)
- **Дистрибутив Linux**: Alpine Linux 3.22.1
- **Архитектура**: ARM64 (aarch64)
- **C библиотека**: musl libc 1.2.5
- **Компилятор**: GCC 14.2.0 (Alpine)
- **Дата тестирования**: 9 августа 2024

## Ключевые различия: Alpine Linux vs Стандартный Linux

### musl libc vs glibc

Alpine Linux использует **musl libc** вместо **glibc**, что обеспечивает:
- Меньшие размеры бинарных файлов
- Лучшие функции безопасности
- Уменьшенную поверхность атаки
- Более быстрый запуск приложений
- Лучшую поддержку статической линковки

### Управление пакетами

- **Менеджер пакетов**: `apk` (Alpine Package Keeper)
- **Формат пакетов**: `.apk` файлы
- **Репозиторий**: Alpine Linux package repository

## Процесс сборки

Бот был успешно собран с использованием следующего процесса:

```bash
# Обновление индекса пакетов
apk update

# Установка зависимостей
apk add build-base cmake boost-dev openssl-dev gtest-dev git nlohmann-json

# Конфигурация с CMake
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release

# Сборка проекта
make -j4
```

## Инструкции по развертыванию

### Для систем Alpine Linux

```bash
# Обновление системы
apk update

# Установка зависимостей
apk add build-base cmake boost-dev openssl-dev gtest-dev git nlohmann-json

# Клонирование и сборка
git clone <repository-url>
cd NeoZorK3
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)

# Запуск бота
./solana_arbitrage_bot --help
```

### Docker Alpine образ

```dockerfile
FROM alpine:3.22

# Установка зависимостей
RUN apk update && apk add --no-cache \
    build-base \
    cmake \
    boost-dev \
    openssl-dev \
    gtest-dev \
    git \
    nlohmann-json

# Сборка бота
WORKDIR /app
COPY . .
RUN mkdir build && cd build && \
    cmake .. -DCMAKE_BUILD_TYPE=Release && \
    make -j$(nproc)

# Запуск бота
CMD ["./build/solana_arbitrage_bot"]
```

1. **Отсутствующие зависимости**
   ```bash

   cmake .. -DCMAKE_BUILD_TYPE=Release
   make VERBOSE=1
   ```

3. **Ошибки выполнения**
   ```bash
   # Проверка зависимостей библиотек
   ldd ./solana_arbitrage_bot
   ```

4. **Совместимость с musl libc**
   ```bash
   # Проверка использования musl
   file ./solana_arbitrage_bot
   # Должно показывать: interpreter /lib/ld-musl-aarch64.so.1
   ```

### Проблемы специфичные для musl libc

1. **Разрешение символов**: musl может иметь разные имена символов
2. **Интерфейс системных вызовов**: Прямые системные вызовы vs обертки glibc
3. **Модель многопоточности**: Немного другая реализация многопоточности
4. **Поддержка локалей**: Другая обработка локалей
