# 🚀 Быстрый старт реальной торговли

## ⚠️ ВАЖНОЕ ПРЕДУПРЕЖДЕНИЕ
**Торговля с реальными деньгами сопряжена с высокими рисками!**
- Всегда начинайте с небольших сумм
- Тщательно тестируйте все настройки
- Используйте только те средства, потерю которых можете себе позволить

## 📋 Быстрые шаги

### 1. Подготовка кошелька
```bash
# Установка Solana CLI
sh -c "$(curl -sSfL https://release.solana.com/stable/install)"

# Создание торгового кошелька
solana-keygen new --outfile ~/.config/solana/trading-wallet.json

# Проверка баланса (нужно минимум 0.1 SOL)
solana balance
```

### 2. Настройка конфигурации
```bash
# Копирование production конфигурации
cp config.production.json config.json

# Редактирование конфигурации
nano config.json

# Замените:
# - "YOUR_PRIVATE_KEY_HERE" на ваш приватный ключ
# - RPC endpoint на платный (рекомендуется)
```

### 3. Тестирование в dry-run режиме
```bash
# Установка dry_run: true в config.json
# Запуск тестирования
./start_real_trading.sh

# Должно работать минимум 24 часа без ошибок
```

### 4. Запуск с реальными деньгами
```bash
# Установка dry_run: false в config.json
# Запуск реальной торговли
./start_real_trading.sh
```

## 🔧 Настройка приватного ключа

### Вариант 1: Переменная окружения

```bash
export SOLANA_PRIVATE_KEY="your_private_key_here"
```

### Вариант 2: .env файл

```bash
echo "SOLANA_PRIVATE_KEY=your_private_key_here" > .env
```

## 📊 Рекомендуемые настройки для начала

### Консервативные настройки (для новичков)
```json
{
  "max_trade_size": 5.0,
  "min_profit_percentage": 1.5,
  "risk_limits": {
    "max_daily_loss": 10.0,
    "max_drawdown": 2.0
  }
}
```

### Агрессивные настройки (для опытных)
```json
{
  "max_trade_size": 50.0,
  "min_profit_percentage": 0.8,
  "risk_limits": {
    "max_daily_loss": 50.0,
    "max_drawdown": 5.0
  }
}
```

## 🛡️ Управление рисками

### Обязательные лимиты
- `max_daily_loss`: Максимальный дневной убыток
- `max_drawdown`: Максимальная просадка
- `max_position_size`: Максимальный размер позиции
- `emergency_stop_loss`: Экстренная остановка

### Мониторинг
```bash
# Просмотр логов в реальном времени
tail -f logs/trading.log

# Проверка статистики
./build/solana_arbitrage_bot --show-stats
```

## 🆘 Экстренная остановка


## 📈 Оптимизация производительности

### 1. RPC Endpoint
- **Бесплатный**: `https://api.mainnet-beta.solana.com` (ограничен)
- **Платный**: QuickNode, Alchemy, Helius (рекомендуется)

### 2. Размер сделок
- Начинайте с малых сумм (1-5 USD)
- Постепенно увеличивайте при успехе

### 3. Частота сделок
- Не более 3-5 сделок в минуту
- Мониторьте комиссии

## 🔍 Отладка проблем

### Частые проблемы
1. **Недостаточно SOL** → Пополните кошелек
2. **RPC ошибки** → Переключитесь на платный RPC
3. **Высокий slippage** → Уменьшите размер сделок
4. **Частые ошибки** → Увеличьте timeout

### Логи для анализа
```bash
# Ошибки
grep "ERROR" logs/trading.log

# Прибыльные сделки
grep "PROFIT" logs/trading.log

# Убыточные сделки
grep "LOSS" logs/trading.log
```

## 📞 Поддержка

При проблемах:
1. Проверьте логи: `tail -f logs/trading.log`
2. Проверьте баланс: `solana balance`
3. Проверьте конфигурацию: `python3 -m json.tool config.json`
4. Обратитесь к документации: `docs/REAL_TRADING_SETUP.md`
